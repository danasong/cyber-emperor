# 赛博皇帝（Cyber Emperor）

`cyber-emperor` 是一个面向 OpenClaw 的实战型 skill，用“三省六部”的组织思路来编排复杂任务。

它不是简单地“多开几个分身”，而是把复杂工作拆成：
- 任务规划
- 风险审查
- 分身派工
- 并行执行
- 统一验收
- 最终交付

## 仓库内容

- `README.md`：中英双语入口
- `README.zh-CN.md`：中文说明
- `README.en.md`：English documentation
- `skills/cyber-emperor/SKILL.md`：技能主文档
- `skills/cyber-emperor/README.md`：skill 辅助说明

## 核心定位

`cyber-emperor` 是一个**治理 / 编排框架**：
- **中书省**：负责任务拆分与方案设计
- **门下省**：负责审查、纠偏、风控
- **尚书省**：负责调度、汇总、收口
- **六部**：负责角色分配、资源边界、规范统一、并行执行、质量验收、文档交付

它与以下能力配合使用：
- `sessions_spawn`：真实派发分身
- `shadow-clone`：并行执行阵型
- `claude-code-hook`：复杂编码零轮询执行

## 适用场景

适合：
- 中大型任务
- 多模块 / 多阶段协作
- 需要“分析 → 实现 → 测试 → 文档 → 交付”的任务
- 复杂编码、项目级推进、统一验收

不适合：
- 单文件小修
- 纯查询
- 十分钟内能直接完成的任务
- 拆分收益低于沟通成本的任务

## 设计原则

1. 本体负责编排，不陷入细碎执行
2. 分身必须真实派发，不能口头模拟多 Agent
3. 复杂编码必须优先接入 `claude-code-hook`
4. 不允许多个分身同时修改同一核心文件
5. 必须有验收层与统一交付层

## 公开发布规则
